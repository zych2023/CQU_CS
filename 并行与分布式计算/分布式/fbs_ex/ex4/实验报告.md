## 3.2 算法设计

### 3.2.1 问题描述

稠密矩阵向量乘法（DMV）的计算公式为：**y = Ax**，其中 **A** 是形状为 m×n 的矩阵，**x** 是 n 维向量，结果 **y** 是 m 维向量。具体计算为：

```
y[i] = Σ(j=0 to n-1) A[i][j] * x[j]    (i = 0, 1, ..., m-1)
```

本实验中矩阵规模为 m=7680, n=42000，矩阵 A 为下三角矩阵（A[i][j]=j+1 当 j≤i，否则为0），向量 x 的所有元素均为1。

### 3.2.2 按行并行算法

**核心思想**：将矩阵 A 按行划分为多个行块，每个MPI进程负责若干行的计算，进程内部使用OpenMP多线程加速各行的点乘运算。

**数据划分**：矩阵 A 的 m 行被均匀分配给 comm_sz 个进程，每个进程处理 my_row = m/comm_sz 行。

**并行流程**：
1. 0号进程生成完整的矩阵 A 和向量 x
2. 通过 `MPI_Bcast` 广播矩阵维度 m、n 和向量 x 到所有进程
3. 通过 `MPI_Scatter` 将矩阵 A 的各行块分发给对应进程
4. 每个进程使用OpenMP 2线程并行计算所分配行的结果
5. 通过 `MPI_Gather` 收集所有进程的计算结果到0号进程

### 3.2.3 按列并行算法

**核心思想**：将矩阵 A 按列划分为多个列块，每个列块与对应部分的向量 x 做乘法，得到部分结果向量，最后对所有部分结果求和得到最终结果。

**数据划分**：先对矩阵 A 进行转置，将 n 列均匀分配给 comm_sz 个进程，每个进程处理 my_col = n/comm_sz 列。

**并行流程**：
1. 0号进程生成矩阵 A 和向量 x，并对 A 进行转置（方便按列分发数据）
2. 通过 `MPI_Bcast` 广播矩阵维度 m、n
3. 通过 `MPI_Scatter` 将转置后的列块分发给各进程，同时分发向量 x 对应的部分
4. 每个进程使用OpenMP 2线程计算该列块对结果向量的贡献
5. 通过 `MPI_Reduce` 使用 MPI_SUM 操作将所有进程的部分结果归约求和到0号进程

## 3.3 源程序代码

### 3.3.1 按行并行源程序（row.cpp）

```cpp
#include<iostream>
#include<mpi.h>
#include<omp.h>
#include<cstring>

using namespace std;
double start_time, end_time;

// 数据生成函数：生成下三角矩阵A和全1向量x
void getData(int*& A, int*& x, int& m, int& n) {
    m = 7680, n = 42000;
    cout << "shape: " << m << " * " << n << endl;
    A = new int[m * n];
    x = new int[n];

    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (j <= i)
                A[i * n + j] = j + 1;   // 下三角部分赋值j+1
            else
                A[i * n + j] = 0;        // 上三角部分为0
        }
    }
    for (int i = 0; i < n; i++) x[i] = 1; // 向量x全为1
}

int main(int argc, char* argv[]) {

    int my_rank, comm_sz;
    int namelen;
    char processor_name[MPI_MAX_PROCESSOR_NAME];
    MPI_Init(NULL, NULL);
    MPI_Comm_rank(MPI_COMM_WORLD, &my_rank);
    MPI_Comm_size(MPI_COMM_WORLD, &comm_sz);
    MPI_Get_processor_name(processor_name, &namelen);

    int* A = nullptr, * x = nullptr, m, n, * result = nullptr, my_row;

    // 0号进程生成数据并开始计时
    if (my_rank == 0) {
        start_time = MPI_Wtime();
        getData(A, x, m, n);
    }
    // 输出各进程所在的节点名称
    cout << processor_name << endl;

    // 广播矩阵维度到所有进程
    MPI_Bcast(&m, 1, MPI_INT, 0, MPI_COMM_WORLD);
    MPI_Bcast(&n, 1, MPI_INT, 0, MPI_COMM_WORLD);

    // 非0号进程分配向量x的内存
    if (my_rank != 0)
        x = new int[n];
    // 广播向量x到所有进程
    MPI_Bcast(x, n, MPI_INT, 0, MPI_COMM_WORLD);

    // 计算每个进程分配的行数
    my_row = m / comm_sz;
    int* local_A = new int[my_row * n];       // 本地矩阵块
    int* local_result = new int[my_row];       // 本地计算结果
    memset(local_result, 0, my_row * sizeof(int));

    // 将矩阵A按行分发给各进程
    MPI_Scatter(A, my_row * n, MPI_INT, local_A, my_row * n,
        MPI_INT, 0, MPI_COMM_WORLD);
    if (my_rank == 0) {
        free(A);
        result = new int[m];
    }

    // 使用OpenMP 2线程并行计算各行的矩阵向量乘法
    # pragma omp parallel for num_threads(2)
    for (int i = 0; i < my_row; i++) {
        for (int j = 0; j < n; j++) {
            local_result[i] += local_A[i * n + j] * x[j];
        }
    }

    // 收集所有进程的计算结果到0号进程
    MPI_Gather(local_result, my_row, MPI_INT, result, my_row,
        MPI_INT, 0, MPI_COMM_WORLD);

    // 0号进程输出结果
    if (my_rank == 0) {
        end_time = MPI_Wtime();
        cout << "row parallel cost time: " << end_time - start_time << "s" << endl;
        cout << "result[m-1] = " << result[m - 1] << endl;
    }

    MPI_Finalize();
    return 0;
}
```

**代码关键点说明**：

1. **`getData` 函数**：按照实验给定的数据生成方式，生成一个 7680×42000 的下三角矩阵 A（当 j≤i 时 A[i][j]=j+1，否则为0），以及全1的向量 x。

2. **`MPI_Bcast`**：0号进程将矩阵维度 m、n 广播到所有进程，确保每个进程知道矩阵大小。同时将向量 x 广播到所有进程，因为按行并行中每个进程都需要完整的向量 x 来计算点乘。

3. **`MPI_Scatter`**：将矩阵 A 按行块均匀分发。每个进程接收 my_row 行数据，即 my_row × n 个整数。

4. **OpenMP并行计算**：使用 `#pragma omp parallel for num_threads(2)` 将外层行循环分配给2个OpenMP线程，每个线程独立计算若干行的点乘结果。

5. **`MPI_Gather`**：将各进程的本地计算结果收集到0号进程的 result 数组中，恢复为完整的 m 维结果向量。

6. **时间测量**：使用 `MPI_Wtime()` 在0号进程上测量从数据生成到结果收集的总时间。

### 3.3.2 按列并行源程序（column.cpp）

```cpp
#include<iostream>
#include<mpi.h>
#include<omp.h>
#include<cstring>

using namespace std;
double start_time, end_time;

// 数据生成函数（与按行并行相同）
void getData(int*& A, int*& x, int& m, int& n) {
    m = 7680, n = 42000;
    cout << "shape: " << m << " * " << n << endl;
    A = new int[m * n];
    x = new int[n];

    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (j <= i)
                A[i * n + j] = j + 1;
            else
                A[i * n + j] = 0;
        }
    }
    for (int i = 0; i < n; i++) x[i] = 1;
}

// 矩阵转置函数：将 m×n 的矩阵A转置为 n×m 的矩阵B
void transpose(int*& A, int*& B, int& m, int& n) {
    B = new int[m * n];
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            B[j*m+i] = A[i * n + j];  // B[j][i] = A[i][j]
        }
    }
    free(A);
}

int main(int argc, char* argv[]) {

    int my_rank, comm_sz;
    int namelen;
    char processor_name[MPI_MAX_PROCESSOR_NAME];
    MPI_Init(NULL, NULL);
    MPI_Comm_rank(MPI_COMM_WORLD, &my_rank);
    MPI_Comm_size(MPI_COMM_WORLD, &comm_sz);
    MPI_Get_processor_name(processor_name, &namelen);

    int* A = nullptr, *B = nullptr, * x = nullptr, m, n, * result = nullptr, my_col;

    // 0号进程生成数据，转置矩阵，并开始计时
    if (my_rank == 0) {
        start_time = MPI_Wtime();
        getData(A, x, m, n);
        transpose(A, B, m, n);  // 转置方便按列分发
    }
    cout << processor_name << endl;

    // 广播矩阵维度
    MPI_Bcast(&m, 1, MPI_INT, 0, MPI_COMM_WORLD);
    MPI_Bcast(&n, 1, MPI_INT, 0, MPI_COMM_WORLD);

    // 计算每个进程分配的列数
    my_col = n / comm_sz;
    int* local_A = new int[my_col * m];    // 本地列块（转置后为 my_col × m）
    int* local_x = new int[my_col];        // 本地向量部分
    int* local_result = new int[m];        // 本地部分结果
    memset(local_result, 0, m * sizeof(int));

    // 分发转置后的列块和对应的向量部分
    MPI_Scatter(B, my_col * m, MPI_INT, local_A, my_col * m,
        MPI_INT, 0, MPI_COMM_WORLD);
    MPI_Scatter(x, my_col, MPI_INT, local_x, my_col,
        MPI_INT, 0, MPI_COMM_WORLD);

    if (my_rank == 0) {
        free(B);
        result = new int[m];
    }

    // 使用OpenMP 2线程并行计算列块的部分结果
    # pragma omp parallel for num_threads(2)
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < my_col; j++) {
            local_result[i] += local_A[j * m + i] * local_x[j];
        }
    }

    // 归约求和：将所有进程的部分结果相加
    MPI_Reduce(local_result, result, m, MPI_INT, MPI_SUM, 0, MPI_COMM_WORLD);

    // 0号进程输出结果
    if (my_rank == 0) {
        end_time = MPI_Wtime();
        cout << "column parallel cost time: " << end_time - start_time << "s"<< endl;
        cout << "result[m-1] = "<<result[m-1] << endl;
    }

    MPI_Finalize();
    return 0;
}
```

**代码关键点说明**：

1. **`transpose` 函数**：按列并行的核心预处理步骤。将 m×n 的矩阵 A 转置为 n×m 的矩阵 B，使得原本按列存储的数据变为按行连续存储，便于后续使用 `MPI_Scatter` 按列块分发数据。转置操作为 B[j×m+i] = A[i×n+j]。

2. **`MPI_Scatter` 分发列块**：转置后，矩阵 B 的每行对应原矩阵的一列。每个进程接收 my_col 行（即原矩阵的 my_col 列），同时通过第二个 `MPI_Scatter` 接收向量 x 中对应的 my_col 个元素。

3. **OpenMP并行计算**：外层循环遍历 m 行，内层循环遍历该进程负责的 my_col 列。每个线程计算若干行的部分结果：local_result[i] += local_A[j×m+i] × local_x[j]，这里 local_A[j×m+i] 对应原矩阵的 A[i][j]（转置后的索引）。

4. **`MPI_Reduce`**：与按行并行的 `MPI_Gather` 不同，按列并行使用归约操作 MPI_SUM 将各进程的部分结果向量逐元素求和，得到最终的完整结果向量。这是因为按列并行中每个进程计算的是结果向量的部分贡献，需要累加才能得到正确结果。

### 3.3.3 两种并行方法的对比

| 对比项 | 按行并行（row.cpp） | 按列并行（column.cpp） |
|--------|---------------------|------------------------|
| 划分方式 | 将矩阵按行划分为行块 | 将矩阵按列划分为列块 |
| 预处理 | 无 | 需要矩阵转置 |
| 数据分发 | `MPI_Scatter` 分发行块 | `MPI_Scatter` 分发转置后的列块和对应向量部分 |
| 各进程结果 | 每个进程得到结果向量的部分行 | 每个进程得到结果向量的部分贡献（需累加） |
| 结果收集 | `MPI_Gather` 直接拼接 | `MPI_Reduce` 归约求和 |
| 每个进程需要的向量x | 完整的向量 x | 仅对应的 x 片段 |
| 通信数据量 | 广播完整 x，Scatter 行块 | Scatter 列块和 x 片段 |

## 4.3 结果分析

### 4.3.1 按行并行性能分析

从实验数据可以看出：

- **基准时间**：1进程1线程（不使用OpenMP）时，运行时间为 5.93s。这是单进程纯串行的基准。
- **OpenMP效果**：1进程2线程时，时间从 5.93s 降至 5.82s，加速比约为 1.02。OpenMP 在单进程内带来的加速效果不明显，原因在于矩阵 A 是下三角矩阵，各行的计算量不均匀（越靠后的行非零元素越多），导致OpenMP的负载不均衡；同时2线程的并行度有限，线程创建和同步的开销抵消了部分加速收益。
- **增加进程数的效果**：从1进程到4进程（均使用2线程OpenMP），运行时间从 5.82s 降至 4.67s，加速比约为 1.25。随着MPI进程数增加，每个进程处理的行数减少（从7680行减少到1920行），计算量降低，运行时间相应减少。
- **加速比不理想的原因**：
  1. **通信开销**：MPI进程间的数据分发（`MPI_Scatter`）和结果收集（`MPI_Gather`）需要时间，进程数越多通信开销越大
  2. **内存带宽瓶颈**：矩阵规模为 7680×42000，约占 1.2GB 内存，计算过程中的大量内存访问可能达到内存带宽上限
  3. **计算量分配**：下三角矩阵各行计算量不同，行块之间的负载不完全均衡

### 4.3.2 按列并行性能分析

- **基准时间**：1进程1线程时，运行时间为 42.10s，远高于按行并行的 5.93s。
- **OpenMP效果**：1进程2线程时，时间从 42.10s 降至 41.22s，加速比约为 1.02，与按行并行类似。
- **增加进程数的效果**：从1进程到4进程（均使用2线程OpenMP），运行时间从 41.22s 降至 23.14s，加速比约为 1.78。相比按行并行，按列并行在增加进程数时获得了更显著的加速。
- **运行时间远高于按行并行的原因**：
  1. **矩阵转置开销**：按列并行需要在0号进程上对 7680×42000 的矩阵进行转置操作，这是一个 O(m×n) 的额外计算
  2. **内存访问模式**：按列并行的内层循环中，`local_A[j * m + i]` 的访问模式在内存中不连续（跨步访问），导致缓存命中率低，而按行并行中 `local_A[i * n + j]` 是连续访问
  3. **归约操作开销**：`MPI_Reduce` 需要对 m 维的结果向量进行全局归约求和，通信量为 m 个整数；而按行并行的 `MPI_Gather` 也是收集 m 个整数，通信量相同，但归约操作本身的计算量更大

### 4.3.3 两种并行方法的对比分析

| 分析维度 | 按行并行 | 按列并行 |
|---------|---------|---------|
| 单进程基准时间 | 5.93s | 42.10s |
| 4进程时间 | 4.67s | 23.14s |
| 进程加速比（1→4） | 1.25× | 1.78× |
| OpenMP加速比（1→2线程） | 1.02× | 1.02× |
| 内存访问模式 | 连续访问，缓存友好 | 跨步访问，缓存不友好 |
| 预处理开销 | 无 | 矩阵转置 |
| 结果收集方式 | Gather（直接拼接） | Reduce（归约求和） |

**按行并行优于按列并行的核心原因**：

1. **内存局部性**：按行并行中，每个进程连续访问自己负责的行块，数据在内存中连续存储，CPU缓存能够有效预取数据。而按列并行中，即使经过转置，内层循环的内存访问仍然是跨步的，缓存效率低。

2. **无额外预处理**：按行并行不需要转置操作，直接按行分发即可。按列并行需要先对整个矩阵进行转置，这是一个耗时的操作。

3. **计算效率**：按行并行中每个进程独立计算自己行块的最终结果，无需与其他进程交互。按列并行中每个进程计算的是部分贡献，还需要通过归约操作合并结果。

### 4.3.4 扩展性分析

从实验数据来看，两种方法在进程数从1增加到4时都表现出了一定的并行加速效果，但加速比并不理想（按行并行仅 1.25×，按列并行 1.78×）。可能的原因包括：

1. **计算规模相对较小**：矩阵虽然有 7680×42000 个元素，但每台服务器仅2核，计算资源有限，通信开销占总时间的比例较高
2. **网络带宽限制**：华为鲲鹏云服务器之间通过网络通信，数据传输需要时间
3. **Amdahl定律**：程序中不可并行的部分（数据生成、结果收集、通信）限制了最大加速比
4. **下三角矩阵特性**：矩阵下三角结构导致各行/列计算量不均，影响负载均衡

### 4.3.5 OpenMP线程数设置分析

实验中OpenMP线程数固定为2，这与每台服务器的2核配置相匹配。从实验结果来看，开启2线程OpenMP相比1线程的加速效果有限（约2%），主要原因：

1. 每个MPI进程已经分配了一个节点的计算资源，OpenMP线程共享同一进程的内存空间和计算资源
2. 线程创建和同步的开销在计算量较小时占比明显
3. 下三角矩阵的不均匀计算量导致OpenMP线程间负载不均衡

## 4.4 实验调试过程

### 4.4.1 环境配置调试

在配置华为鲲鹏云服务器环境时，主要解决了以下问题：

1. **MPI安装**：使用 `apt install openmpi-bin openmpi-common libopenmpi-dev` 完成 OpenMPI 的安装，安装后验证 `mpicc --version` 确认可用
2. **主机名配置**：在每台节点上通过 `hostnamectl set-hostname` 设置主机名，并在 `/etc/hosts` 中添加所有节点的IP-主机名映射，确保节点间可以通过主机名互相访问
3. **SSH免密登录**：生成SSH密钥对并将公钥分发到所有节点，实现无密码SSH登录，这是MPI跨节点运行的必要条件

### 4.4.2 编译调试

编译过程中使用 `mpicxx -fopenmp` 命令同时启用 MPI 和 OpenMP 支持。初始编译时遇到的注意事项：
- 必须使用 `mpicxx` 而不是普通的 `g++`，否则会缺少 MPI 库的链接
- `-fopenmp` 选项必须添加，否则 OpenMP 指令会被忽略

### 4.4.3 运行调试

1. **Root用户运行限制**：OpenMPI 默认不允许 root 用户运行 `mpirun`，需要设置环境变量：
   ```bash
   export OMPI_ALLOW_RUN_AS_ROOT=1
   export OMPI_ALLOW_RUN_AS_ROOT_CONFIRM=1
   ```

2. **主机文件配置**：确保 `hostfile` 中的主机名与 `/etc/hosts` 中的配置一致，且每个节点的 `slots=1` 满足实验要求

3. **结果验证**：通过检查 `result[m-1]` 的值（应为 29495040）来验证程序计算的正确性，所有实验运行均得到了正确结果
