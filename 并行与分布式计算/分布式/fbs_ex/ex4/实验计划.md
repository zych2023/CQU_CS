### 已有代码
- `row.cpp`：按行并行实现 ✓ 基本完成

### 代码问题
1. **进程数限制**：当前代码要求 `m % comm_sz == 0` 和 `n % comm_sz == 0`
   - m=7680, n=42000
   - 7680 ÷ 6 = 1280 ✓
   - 42000 ÷ 6 = 7000 ✓
   - 所以进程数1-6都可以整除，没问题

2. **内存管理**：使用 `free()` 释放 `new` 分配的内存，应使用 `delete[]`
   - 但这不会导致程序崩溃，只是不规范

3. **输出格式**：需要添加节点名称输出

## 三、预期结果

### result[m-1] 的值
- m=7680, n=42000
- 矩阵A是下三角矩阵，A[i][j] = j+1 (当 j<=i)
- x[i] = 1
- result[i] = Σ(j=0 to i) (j+1) = (i+1)(i+2)/2
- result[7679] = 7680 × 7681 / 2 = 29,491,200

### 性能分析预期
1. **进程数增加**：理论上运行时间应该减少
2. **OpenMP效果**：2线程应该比1线程快
3. **通信开销**：进程数增加会带来通信开销，可能影响加速比

## 七、快速执行脚本

创建 `run_experiments.sh`：

```bash
#!/bin/bash

echo "=== 实验四：OpenMP与MPI混合编程实验 ==="
echo ""

# 编译
echo "编译代码..."
mpicxx -fopenmp -o row_dmvp row.cpp
mpicxx -fopenmp -o col_dmvp column.cpp

echo ""
echo "=== 按行并行实验 ==="

# 进程数=1，不使用OpenMP
echo "进程数=1, OpenMP=1"
OMP_NUM_THREADS=1 mpirun -np 1 ./row_dmvp

# 进程数=1-6，使用OpenMP
for np in 1 2 3 4 5 6; do
    echo "进程数=$np, OpenMP=2"
    OMP_NUM_THREADS=2 mpirun -np $np ./row_dmvp
done

echo ""
echo "=== 按列并行实验 ==="

# 进程数=1，不使用OpenMP
echo "进程数=1, OpenMP=1"
OMP_NUM_THREADS=1 mpirun -np 1 ./col_dmvp

# 进程数=1-6，使用OpenMP
for np in 1 2 3 4 5 6; do
    echo "进程数=$np, OpenMP=2"
    OMP_NUM_THREADS=2 mpirun -np $np ./col_dmvp
done

echo ""
echo "实验完成！"
```
