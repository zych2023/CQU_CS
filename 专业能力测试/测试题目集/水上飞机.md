# 题目
请设计以下航行器、飞机、船、水上飞机等 4 个类。

- CRAFT 为航行器类，是公共基类，提供航行器的基本特性。包括：
    
    一个保护数据成员：speed(速度)。
    
    三个公有成员函数：构造函数(初始化速度)、析构函数和 Show 函数(显示速度)。
    
- PLANE 为飞机类，以公有方式继承 CRAFT 类，在航行器类的基础上增加飞机的特性。包括：
    
    一个保护数据成员：width(翼展)。
    
    三个公有成员函数：构造函数(初始化速度和翼展)、析构函数和 Show 函数(显示速度和翼展)。
    
- SHIP 为船类，以公有方式继承 CRAFT 类，在航行器类的基础上增加船的特性。包括：
    
    一个保护数据成员：depth(吃水深度)。
    
    三个公有成员函数：构造函数(初始化速度和吃水深度)、析构函数和 Show 函数(显示速度和吃水深度)。
    
- SEAPLANE 为水上飞机类，同时以公有方式继承 PLANE 类和 SHIP 类，兼具飞机和船的特性。包括：
    
    三个公有成员函数：构造函数(初始化速度、翼展、吃水深度)、析构函数和 Show 函数(显示速度、翼展和吃水深度)。

# 解答
```cpp
#include <iostream>
using namespace std;

// 航行器类（虚基类）
class CRAFT {
protected:
    double speed; // 速度
public:
    // 构造函数：初始化速度并输出创建信息
    CRAFT(double s) : speed(s) {
        cout << "创建航行器(速度: " << speed << ")" << endl;
    }
    // 虚析构函数：输出销毁信息
    virtual ~CRAFT() {
        cout << "销毁航行器(速度: " << speed << ")" << endl;
    }
    // 虚函数Show：显示速度
    virtual void Show() {
        cout << "航行(速度: " << speed << ")" << endl;
    }
};

// 飞机类：虚继承自CRAFT
class PLANE : virtual public CRAFT {
protected:
    double width; // 翼展
public:
    // 构造函数：初始化速度和翼展，调用基类构造函数
    PLANE(double s, double w) : CRAFT(s), width(w) {
        cout << "创建飞机(翼展: " << width << ")" << endl;
    }
    // 虚析构函数：输出销毁信息
    virtual ~PLANE() {
        cout << "销毁飞机(翼展: " << width << ")" << endl;
    }
    // 重写Show函数：显示速度和翼展
    void Show() override {
        cout << "航行(速度: " << speed << ", 翼展: " << width << ")" << endl;
    }
};

// 船类：虚继承自CRAFT
class SHIP : virtual public CRAFT {
protected:
    double depth; // 吃水深度
public:
    // 构造函数：初始化速度和吃水深度，调用基类构造函数
    SHIP(double s, double d) : CRAFT(s), depth(d) {
        cout << "创建船(吃水: " << depth << ")" << endl;
    }
    // 虚析构函数：输出销毁信息
    virtual ~SHIP() {
        cout << "销毁船(吃水: " << depth << ")" << endl;
    }
    // 重写Show函数：显示速度和吃水深度
    void Show() override {
        cout << "航行(速度: " << speed << ", 吃水: " << depth << ")" << endl;
    }
};

// 水上飞机类：继承自PLANE和SHIP
class SEAPLANE : public PLANE, public SHIP {
public:
    // 构造函数：初始化速度、翼展、吃水深度，直接调用虚基类CRAFT的构造函数
    SEAPLANE(double s, double w, double d) : CRAFT(s), PLANE(s, w), SHIP(s, d) {
        cout << "创建水上飞机" << endl;
    }
    // 虚析构函数：输出销毁信息
    virtual ~SEAPLANE() {
        cout << "销毁水上飞机" << endl;
    }
    // 重写Show函数：显示速度、翼展和吃水深度
    void Show() override {
        cout << "航行(速度: " << speed << ", 翼展: " << width << ", 吃水: " << depth << ")" << endl;
    }
};
```

# 虚继承
在 C++ 中，**虚继承（Virtual Inheritance）** 是一种特殊的继承方式，用于解决**多继承中的菱形问题（Diamond Problem）**——即当一个派生类间接多次继承自同一个基类时，基类的成员会被多次复制，导致数据冗余和访问歧义。


### 问题场景：菱形继承的隐患
假设存在如下继承关系：
```
     CRAFT（基类）
    /       \
  PLANE    SHIP（中间派生类）
    \       /
     SEAPLANE（最终派生类）
```
如果 `PLANE` 和 `SHIP` 都**普通继承**自 `CRAFT`，那么 `SEAPLANE` 继承时会同时拥有 `PLANE` 和 `SHIP` 各自包含的 `CRAFT` 成员（如 `speed`）。此时：
- `SEAPLANE` 中会有两份 `CRAFT` 的 `speed` 数据（冗余）。
- 访问 `speed` 时，编译器无法确定是来自 `PLANE` 还是 `SHIP`（歧义）。


### 虚继承的作用
虚继承通过让中间派生类（`PLANE`、`SHIP`）**虚继承**自基类（`CRAFT`），确保最终派生类（`SEAPLANE`）中**只保留基类的一份成员副本**，从而解决数据冗余和访问歧义。

在你的代码中：
```cpp
class PLANE : virtual public CRAFT { ... };  // 虚继承
class SHIP : virtual public CRAFT { ... };   // 虚继承
```
`PLANE` 和 `SHIP` 虚继承自 `CRAFT`，因此 `SEAPLANE` 继承时，`CRAFT` 的成员（如 `speed`）只会被初始化一次，且访问时无歧义。


### 虚继承的关键特性
1. **共享基类实例**：虚继承的基类被称为“虚基类”，其成员在最终派生类中只有一份实例，被所有中间派生类共享。
   
2. **构造函数调用规则**：  
   普通继承中，中间派生类会自动调用基类构造函数；但虚继承中，**最终派生类必须直接调用虚基类的构造函数**，中间派生类对虚基类的构造函数调用会被忽略。  
   例如你的 `SEAPLANE` 构造函数：
   ```cpp
   SEAPLANE(double s, double w, double d) : CRAFT(s), PLANE(s, w), SHIP(s, d) { ... }
   ```
   这里直接调用 `CRAFT(s)` 初始化虚基类，而 `PLANE` 和 `SHIP` 构造函数中的 `CRAFT(s)` 会被忽略，避免重复初始化。

3. **解决菱形问题**：通过消除基类成员的多份副本，保证了数据的一致性和访问的明确性。


### 总结
虚继承是 C++ 为解决多继承中“菱形问题”设计的机制，核心是让多个中间派生类共享同一个基类实例，避免数据冗余和访问歧义。你的代码中，`PLANE` 和 `SHIP` 虚继承自 `CRAFT`，确保 `SEAPLANE` 能正确共享 `CRAFT` 的 `speed` 成员，是多继承场景下的规范用法。
