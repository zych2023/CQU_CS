在树的遍历中选择**栈（Stack）** 而非**队列（Queue）**，本质上是由遍历的**顺序需求**决定的。栈的“后进先出（LIFO）”特性与先序、中序、后序遍历的逻辑一致，而队列的“先进先出（FIFO）”特性则对应另一种遍历方式（层序遍历）。具体原因如下：


### 1. 栈的特性适配深度优先遍历（DFS）
树的先序、中序、后序遍历都属于**深度优先遍历（DFS）**：遍历过程中，会沿着一条路径“深入”到最底层节点，再回溯处理其他分支。  
栈的“后进先出”特性天然适合这种“深入+回溯”的逻辑：
- 当访问一个节点后，需要先处理它的子节点（而非其他分支的节点），栈可以暂时保存未处理的节点，优先弹出最新压入的子节点（即“深入”）。
- 例如先序遍历中，压入子节点时逆序压栈，弹出时会按原顺序处理子节点，保证“根→子节点”的顺序。


### 2. 队列的特性适配广度优先遍历（BFS）
队列的“先进先出”特性适合**层序遍历（广度优先遍历）**：遍历过程中，会先处理完当前层的所有节点，再处理下一层节点（按层次扩散）。  
例如：
```cpp
// 层序遍历（用队列）
queue<int> q;
q.push(root);
while (!q.empty()) {
    int u = q.front();
    q.pop();
    result.push_back(u);
    // 正序压入子节点，保证按层次顺序处理
    for (int v : children[u]) {
        q.push(v);
    }
}
```
此时队列会按“根节点→第一层子节点→第二层子节点...”的顺序处理，与栈的使用场景完全不同。


### 3. 为什么先序遍历不能用队列？
如果用队列实现先序遍历（以之前的示例树为例）：
```
        1
      / | \
     2  3  4
    / \
   5   6
```
- 队列初始：`[1]`，弹出1并加入结果，然后正序压入子节点 `[2,3,4]`，队列变为 `[2,3,4]`。
- 弹出2并加入结果，压入子节点 `[5,6]`，队列变为 `[3,4,5,6]`。
- 弹出3并加入结果（无子女），队列变为 `[4,5,6]`。
- 弹出4并加入结果（无子女），队列变为 `[5,6]`。
- 最终结果为 `[1,2,3,4,5,6]`，这是**层序遍历结果**，而非先序遍历的 `[1,2,5,6,3,4]`。

可见，队列会导致遍历顺序按“层次”扩散，而非“深度优先”，不符合先序遍历的需求。


### 总结
- **栈**用于深度优先遍历（先序、中序、后序），利用“后进先出”特性实现“深入+回溯”。
- **队列**用于广度优先遍历（层序），利用“先进先出”特性实现“按层次扩散”。

选择哪种数据结构，取决于遍历需要的顺序——栈是深度优先的“天然搭档”，而队列是广度优先的“天然搭档”。
