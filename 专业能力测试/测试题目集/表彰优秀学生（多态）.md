# 题目
学期结束，班主任决定表彰一批学生，已知该班学生数在6至50人之间，有三类**学生**：*普通生，特招运动员，学科专长生*，其中学科专长生不超过5人。

主函数根据输入的信息，相应建立GroupA, GroupB, GroupC类对象。

GroupA类是普通生，有2门课程的成绩（均为不超过100的非负整数）；

GroupB类是特招运动员，有2门课程的成绩（均为不超过100的非负整数），1次运动会的表现分，表现分有：A、B、C、D共4等。

GroupC类是学科专长生，有5门课程的成绩（均为不超过100的非负整数）。

表彰人员至少符合以下3个条件中的一个：

（1）2门课程平均分在普通生和特招运动员中，名列第一者。

a.该平均分称为获奖线。

b.存在成绩并列时，则全部表彰，例如某次考试有2人并列第1，则他们全部表彰。

（2）5门课程平均分达到或超过获奖线90%的学科专长生，给予表彰。

（3）2门课程平均分达到或超过获奖线70%的特招运动员，如果其运动会表现分为A，给予表彰。

输入格式：每个测试用例占一行，第一项为类型，1为普通生，2为特招运动员，3为学科专长生, 输入0表示输入的结束。**第二项是学号，第三项是姓名**。对于普通生来说，共输入5项，第4、5项是课程成绩。对于特招运动员来说，共输入6项，第4、5项是课程成绩，第6项是运动会表现。对于学科专长生来说，共输入8项，第4、5、6、7、8项是课程成绩。

输出时，打印要表彰的学生的学号和姓名。(输出顺序与要表彰学生的输入前后次序一致)

# 解答

```cpp
double awardLine = -1; // 全局变量：获奖线

class Student {
protected:
    string num;
    string name;
public:
    Student(string n, string na):num(n),name(na){}
    virtual ~Student(){}
    virtual double getAvg() const { return 0; }
    virtual bool isAwarded() const = 0;
    virtual void display() const {
        if(isAwarded()) {
            cout << num << " " << name << endl;
        }
    }
};

// GroupA 普通生
class GroupA: public Student {
    int s1,s2;
public:
    GroupA(string n,string na,int a,int b):Student(n,na),s1(a),s2(b){
        awardLine = max(awardLine, getAvg()); // 更新获奖线
    }
    double getAvg() const override { return (s1+s2)/2.0; }
    bool isAwarded() const override {
        return getAvg() == awardLine;
    }
};

// GroupB 特招运动员
class GroupB: public Student {
    int s1,s2;
    char gs;
public:
    GroupB(string n,string na,int a,int b,char g):Student(n,na),s1(a),s2(b),gs(g){
        awardLine = max(awardLine, getAvg()); // 更新获奖线
    }
    double getAvg() const override { return (s1+s2)/2.0; }
    bool isAwarded() const override {
        double avg = getAvg();
        if(avg == awardLine) return true;             // 条件1
        if(avg >= awardLine*0.7 && gs=='A') return true; // 条件3
        return false;
    }
};

// GroupC 学科专长生
class GroupC: public Student {
    int s1,s2,s3,s4,s5;
public:
    GroupC(string n,string na,int a,int b,int c,int d,int e):Student(n,na),s1(a),s2(b),s3(c),s4(d),s5(e){}
    double getAvg() const { return (s1+s2+s3+s4+s5)/5.0; }
    bool isAwarded() const override {
        return getAvg() >= awardLine*0.9; // 条件2
    }
};
```


# 继承与多态
### C++ 继承与多态详解（结合示例代码）
#### 一、继承（Inheritance）
继承是面向对象编程的核心特性之一，它允许一个类（子类/派生类）继承另一个类（父类/基类）的属性和方法，从而实现代码复用和层次化设计。

在示例中：
- `Student` 是**基类**（父类），定义了所有学生的共同属性（`num` 学号、`name` 姓名）和行为（`getAvg` 求平均分、`isAwarded` 判断是否获奖、`display` 显示信息）。
- `GroupA`、`GroupB`、`GroupC` 是**派生类**（子类），通过 `public Student` 继承了 `Student` 的所有成员，并根据自身特点扩展了属性和重写了方法：
  - `GroupA`（普通生）新增 `s1`、`s2` 两门成绩；
  - `GroupB`（特招运动员）新增 `s1`、`s2` 成绩和 `gs` 等级；
  - `GroupC`（学科专长生）新增 `s1-s5` 五门成绩。


#### 二、**多态（Polymorphism）**
多态是指**同一接口（方法）在不同对象上表现出不同行为**，C++ 中通过**虚函数（virtual function）** 和**动态绑定**实现。

示例中多态的体现：
1. **虚函数与重写（override）**  
   基类 `Student` 中：
   - `virtual double getAvg() const`：虚函数，声明为 `virtual` 表示允许派生类重写；
   - `virtual bool isAwarded() const = 0`：纯虚函数（`=0`），基类不实现，强制派生类必须重写（此时 `Student` 为**抽象类**，无法实例化）。

   派生类通过 `override` 关键字明确重写基类虚函数：
   - `GroupA::getAvg`、`GroupB::getAvg`、`GroupC::getAvg` 分别计算不同类型学生的平均分；
   - `isAwarded` 则根据不同规则判断是否获奖（核心多态行为）。


2. **动态绑定（运行时多态）**  
   当使用**基类指针/引用**指向派生类对象时，调用虚函数会自动匹配到派生类的重写版本（而非基类版本），这一过程在运行时确定。

   例如：
   ```cpp
   Student* s1 = new GroupA("001", "Alice", 90, 80);
   Student* s2 = new GroupB("002", "Bob", 70, 60, 'A');
   s1->isAwarded();  // 调用 GroupA::isAwarded
   s2->isAwarded();  // 调用 GroupB::isAwarded
   ```


#### 三、示例代码关键细节解析
1. **访问控制**  
   基类 `Student` 的 `num` 和 `name` 声明为 `protected`，表示：
   - 基类自身可访问；
   - 派生类可直接访问（无需通过 `public` 方法）；
   - 类外不可访问（保证封装性）。


2. **纯虚函数与抽象类**  
   `isAwarded() = 0` 使 `Student` 成为抽象类，无法创建实例（如 `Student s("003", "Charlie");` 会报错），只能作为基类被继承，强制派生类实现 `isAwarded` 逻辑，确保所有学生类型都有明确的获奖判断规则。


3. **虚析构函数**  
   基类析构函数声明为 `virtual ~Student()`，确保删除基类指针指向的派生类对象时，会先调用派生类析构函数，再调用基类析构函数，避免内存泄漏。


4. **多态行为示例**  
   - `GroupA::isAwarded`：仅平均分等于全局获奖线（`awardLine`）时获奖；
   - `GroupB::isAwarded`：满足以下任一条件获奖：平均分等于获奖线，或平均分≥获奖线70%且等级为'A'；
   - `GroupC::isAwarded`：平均分≥获奖线90%时获奖。

   当通过 `Student*` 调用 `isAwarded` 时，会根据对象实际类型（`GroupA` / `GroupB` / `GroupC`）执行不同逻辑，体现多态。


#### 四、总结
- **继承**：通过 `class 派生类 : 继承方式 基类` 实现代码复用，建立类的层次关系；
- **多态**：通过 `virtual` 虚函数和 `override` 重写，使基类接口在派生类中表现出不同行为，核心是**运行时动态绑定**；
- 示例中 `Student` 作为抽象基类定义统一接口，派生类各自实现具体逻辑，既保证了代码扩展性（新增学生类型只需继承 `Student` 并重写方法），又通过多态简化了调用（统一用基类指针操作）。
